What is this algorithm?

This is an algorithm that creates a Predator-Prey simulation. This is a simulation known as cellular automaton. Basically, a 2D grid of cells go through several iterations over time according to a certain set of rules that determine each cell's next state.

Okay, then what are the rules?

Here's an image showing a cell surrounded by its visible neighbors. There are a couple of ways a cell can change state and they are as follows:

  • A predator cell will always move if it's able. It moves randomly and has a certain amount of health that decays by 1 with each iteration. If the health of the cell reaches 0 then it will die. The only way for predators to gain health is to eat a prey cell. Upon eating it, the predator cell will gain all of that prey's health. If the predator's health reaches a pre-defined threshold, it will birth an offspring in the cell from which it moved. The offspring and the parent cell will then start with their health equal to that of the threshold value.
  • A prey cell will always move if it's able. It moves randomly and has a certain amount of health that increases by 1 with each iteration. If the health of the cell ever reaches a pre-defined threshold, the cell will produce and offspring in the cell from which it moved. The offspring and the parent cell will then start with a random amount of health up to half of the threshold value. The only way for a prey cell to die is for it to be eaten by a predator cell.

The interesting thing about this simulation is the steady-state it reaches of a balance between predator and prey. When plotting the populations against each other, a circle (or at least a decently round looking shape) is formed as the species' populations are dependent on each other. If the predators all die out, the prey will grow until they take up the whole area. If all of the prey die, the predators will die out until none are left. Neither of these situations are particularly interesting to see so I gave the user the option of preventing this outright. If there is only one prey, the predators will not eat it and if there is only one predator, it will not die out. This provides the benefit of making every starting position result in an interesting steady-state. I recommend you play around with the values at your disposal on the project page.

In terms of actually making all of this happen, I just go through the grid cell-by-cell and decide what to do with it according to the rules above. Each cell has a property to tell if it's already been moved this iteration so that we don't repeat anything.
