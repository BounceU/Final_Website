What is this algorithm?

This is an algorithm called Conway's Game of Life. This is a simulation known as cellular automaton. Basically, a 2D grid of cells go through several iterations over time according to a certain set of rules that determine each cell's next state.

Okay, then what are the rules?

Here's an image showing a cell surrounded by its visible neighbors. There are a couple of ways a cell can change state and they are as follows:

  1. An inactive cell will turn active if it has exactly 3 active neighbors.
  2. An active cell will stay active if it has 2 or 3 active neighbors.
  3. An active cell will turn inactive if it has less than 2 or more than 3 active neighbors.

Okay cool, but how do you actually apply those rules?

Well, the structure is as follows: every iteration we go through the grid (an array of integers) left-to-right, top-to-bottom. For each cell, we look at the 8 cells around it and count the amount of them that are active. We then decide whether the cell will live or die in the next iteration. We can't just switch these from 0 to 1 and vice versa, though. We need to calculate the next state for the whole grid before we can change any of the elements. The most simple way to accomplish this would be to use another grid but that uses more memory and would be a little bit slower. Instead, we use only 1 grid but define multiple values instead of being 0-inactive, 1-active. We keep those values but define 2-inactive now but active next iteration, 3-active now but inactive next iteration. Then when we're drawing the grid to the screen, as we go through each element to draw it we turn all of the 2's to 1's and all of the 3's to 0's. This fixes the grid in time for the next calculations.

That's it, this algorithm is fairly simple.
